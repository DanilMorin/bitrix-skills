# Проведи Bitrix-ready аудит проекта

Используй правила из папки `docs/`.

Проект сейчас является статической HTML/SCSS/JS-вёрсткой. Не переноси проект на PHP и не создавай Bitrix-шаблон.

Задача:

1. Проанализируй структуру проекта.
2. Найди HTML, SCSS и JS файлы.
3. Найди повторяющиеся блоки.
4. Определи будущие Bitrix-сущности.
5. Найди проблемы для будущей посадки.
6. Предложи безопасные изменения.
7. Раздели рекомендации по риску: low, medium, high.
8. Не меняй код без отдельного разрешения.

Формат ответа возьми из `templates/audit-report.template.md`.
