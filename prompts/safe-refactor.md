# Предложи безопасный рефакторинг

Используй результаты аудита.

Предложи только те изменения, которые не должны сломать GitHub Pages preview.

Не делай:

- перевод на PHP;
- массовое переименование классов;
- изменение сборки;
- замену HTML на Bitrix-компоненты.

Для каждого изменения укажи:

- файл;
- что изменить;
- зачем;
- риск;
- как проверить.
