# HTML-правила для Bitrix-ready вёрстки

## Основные правила

- Использовать семантичные теги.
- Не использовать inline-style без необходимости.
- Не дублировать HTML ради визуала, если можно решить SCSS.
- Изображения должны иметь alt.
- Ссылки должны быть понятными и готовыми к замене на динамические.
- Повторяемые элементы должны иметь стабильную структуру.
- Классы должны быть понятны другому разработчику.

## Нельзя

- Завязывать HTML на случайный порядок элементов.
- Использовать слишком общие классы без контекста.
- Смешивать несколько независимых блоков в один огромный блок.
- Делать разные HTML-структуры для одинаковых карточек без причины.

## Желательно

- Использовать БЭМ.
- Добавлять модификаторы для вариантов.
- Делать блоки переносимыми.
- Отделять контент от визуальной логики.
