# JS-правила для Bitrix-ready вёрстки

## Основные правила

- Проверять существование элементов перед навешиванием обработчиков.
- Не использовать глобальные переменные без необходимости.
- Не завязывать логику на конкретное количество карточек.
- Не завязывать логику на случайный порядок DOM-элементов.
- Использовать data-атрибуты для поведения.
- Делать повторную инициализацию возможной.
- Не смешивать бизнес-логику и визуальное поведение.

## Почему это важно

В Bitrix элементы могут выводиться динамически через компоненты, ajax, фильтры и постраничную навигацию.

JS должен работать не только на статичном HTML, но и на HTML, который пришёл из шаблона компонента.
