# SCSS-правила для Bitrix-ready вёрстки

## Основные правила

- Не превышать 3–4 уровня вложенности.
- Не писать стили на глобальные теги без необходимости.
- Не использовать хаотичные селекторы.
- Повторяющиеся значения выносить в переменные или миксины.
- Компонентные стили держать рядом по смыслу.
- Не ломать адаптивность.
- Не полагаться на порядок случайных CSS-файлов.

## Рискованные классы

Осторожно использовать классы:

- container;
- row;
- col;
- card;
- btn;
- nav;
- active;
- modal;
- dropdown.

Они могут конфликтовать с Bootstrap или стилями Bitrix.
