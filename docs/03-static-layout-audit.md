# Аудит статической вёрстки перед Bitrix

## Цель

Найти проблемы, которые усложнят будущую посадку на 1С-Битрикс, но не ломать текущий статический проект.

## Проверить

- HTML-структуру.
- Повторяемые блоки.
- Inline-style.
- Alt у изображений.
- Семантику.
- Ссылки.
- JS-обработчики.
- SCSS-вложенность.
- Адаптивность.
- Глобальные классы.
- Возможные конфликты с Bootstrap.
- Возможность разделить страницу на header, work_area и footer.

## Результат аудита

AI должен выдать:

1. Общую оценку.
2. Таблицу повторяющихся блоков.
3. Проблемы.
4. Безопасные правки.
5. Риски.
6. Карту будущей Bitrix-архитектуры.
