# Безопасный рефакторинг

## Low risk

Можно делать сейчас:

- убрать inline-style в SCSS;
- добавить alt;
- добавить data-атрибуты;
- сгруппировать SCSS;
- добавить комментарии;
- добавить модификаторы без удаления старых классов;
- вынести повторяемые значения в переменные;
- улучшить JS-проверки на существование элементов.

## Medium risk

Делать после визуальной проверки:

- объединять похожие блоки;
- менять HTML-структуру;
- переносить JS в отдельные файлы;
- переименовывать второстепенные классы;
- менять структуру SCSS.

## High risk

Оставить до посадки:

- перевод HTML в PHP;
- замена статичных карточек на Bitrix-компоненты;
- изменение сборки;
- массовое переименование классов;
- изменение сетки;
- удаление визуально важных дублей.
